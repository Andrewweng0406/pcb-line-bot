# AWS 部署前檢查清單

在部署到生產環境之前，請完成以下所有檢查項目。

---

## 📋 本機驗證

### 應用功能測試
- [ ] 應用能正確啟動（`docker-compose up`）
- [ ] 健康檢查端點可訪問（`curl http://localhost:8000/health`）
- [ ] API 文檔可訪問（`http://localhost:8000/docs`）
- [ ] 報價計算正常工作
- [ ] 資料庫查詢返回正確結果
- [ ] 日誌文件正確生成於 `logs/` 目錄

### LINE Bot 功能測試
- [ ] Webhook 通過 ngrok 正常工作
- [ ] 文字消息解析正確
- [ ] 圖片上傳和識別工作正常
- [ ] 報價單匯出生成成功
- [ ] 查詢歷史報價功能工作正常

### 環境配置驗證
- [ ] `.env.example` 中所有必需變數已設置
- [ ] `LINE_CHANNEL_ACCESS_TOKEN` 和 `LINE_CHANNEL_SECRET` 已驗證
- [ ] `OPENAI_API_KEY` 已驗證和測試
- [ ] 資料庫連接字符串正確
- [ ] 無硬編碼的機密信息在代碼中

---

## 🔒 安全性檢查

### 代碼安全
- [ ] 敏感信息不在代碼中（檢查 git history）
- [ ] `.gitignore` 包含所有敏感文件（.env、*.db、logs/）
- [ ] 沒有 `print()` 或 `pprint()` 輸出敏感數據
- [ ] SQL 注入防護已驗證（使用 SQLAlchemy ORM）
- [ ] 輸入驗證已實施

### 依賴安全
- [ ] 運行 `pip check` 驗證沒有依賴衝突
- [ ] 檢查依賴是否有已知漏洞（使用 `pip install safety` 和 `safety check`）
- [ ] 更新到最新的依賴版本（如果安全）

### AWS 安全
- [ ] 創建了專用的 IAM 用戶用於部署（不使用根帳戶）
- [ ] IAM 政策遵循最小權限原則
- [ ] AWS 金鑰存儲在安全位置（不在代碼中）
- [ ] 啟用了 MFA（多因素認證）
- [ ] CloudTrail 已啟用用於審計

---

## 📊 性能和可擴展性檢查

### 資料庫配置
- [ ] 連接池大小正確配置（當前設置為 10）
- [ ] 資料庫索引已創建在常用查詢字段上
- [ ] 執行過 VACUUM 和 ANALYZE（如果使用 PostgreSQL）
- [ ] 備份策略已配置（RDS 自動備份設置為 7 天）
- [ ] 資料庫參數針對預期的工作量優化

### Redis 配置
- [ ] Redis 記憶 TTL 已設置為合理的時間（預設 24 小時）
- [ ] 故障轉移機制已測試（Redis 不可用時回退到本地）
- [ ] Redis 持久化已配置

### S3 配置
- [ ] S3 bucket 已創建和配置
- [ ] Bucket 策略設置為私有
- [ ] 生命週期規則已配置自動刪除舊導出
- [ ] CORS 配置（如需要）
- [ ] 版本控制已啟用

### ECS 配置
- [ ] Fargate 任務 CPU 和記憶設置合理（256MB - 512MB CPU）
- [ ] 初始所需計數設置為 2（高可用）
- [ ] 最大計數設置為 10（成本控制）
- [ ] 健康檢查設置正確（30 秒間隔）

## 🔄 部署流程

按順序執行：

1. [ ] `git push` 提交所有更改
2. [ ] 構建 Docker 映像並推送到 ECR
3. [ ] 驗證映像在 ECR 中可用
4. [ ] 執行 CloudFormation 堆棧創建
5. [ ] 等待堆棧創建完成（10-15 分鐘）
6. [ ] 驗證所有資源已創建
7. [ ] 更新 LINE Webhook URL
8. [ ] 執行煙霧測試（smoke test）
9. [ ] 監控日誌 24 小時
10. [ ] 設置監控告警和通知

## 回滾計劃

如果部署後出現重大問題：

1. 檢查日誌以確定問題：
   ```bash
   aws logs tail /ecs/pcb-bot --follow
   ```

2. 回滾到前一個版本：
   ```bash
   aws ecs update-service \
     --cluster pcb-bot-cluster \
     --service pcb-bot-service \
     --force-new-deployment
   ```

3. 更新 LINE Webhook 指向舊環境（如需要）

4. 調查和修復問題

5. 重新部署新版本
